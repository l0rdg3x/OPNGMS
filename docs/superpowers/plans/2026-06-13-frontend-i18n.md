# Frontend UI Localization (i18n) — Implementation Plan

**Goal:** Localize the frontend UI into en/it/es/fr/de/pt/nl, switchable per operator and
persisted in localStorage, with no backend changes.

**Architecture:** Keep the custom React Context i18n. Widen `Dict` so sibling dictionaries
enforce key parity at compile time; add a stateful provider with localStorage persistence +
browser detection; add native-name dictionaries and a `LanguageSwitcher`.

**Tech stack:** React 19, TypeScript, Mantine v9, Vitest.

Spec: `docs/superpowers/specs/2026-06-13-frontend-i18n-design.md`.

---

## Task 1 — Widen `Dict` for translatable values

**Files:** Modify `frontend/src/i18n/en.ts`.

- Add `type Localized<T> = { [K in keyof T]: T[K] extends string ? string : Localized<T[K]> };`
  and change `export type Dict = typeof en;` → `export type Dict = Localized<typeof en>;`.
- Verify `tsc -b` still passes with only `en` wired (en is `as const`, assignable to widened
  `Dict`).

## Task 2 — Locale set + metadata

**Files:** Create `frontend/src/i18n/locale.ts`.

- `SUPPORTED_LOCALES = ["en","it","es","fr","de","pt","nl"] as const`, `type Locale`,
  `LOCALE_LABELS` (native names), `DEFAULT_LOCALE = "en"`.
- `detectInitialLocale()`: localStorage `opngms.locale` (if supported) → `navigator.language`
  prefix match → `DEFAULT_LOCALE`; try/catch around storage.
- `persistLocale(locale)`: write localStorage, try/catch.
- `isLocale(x): x is Locale` guard used by detection.

## Task 3 — Stateful provider + `useLocale` + `<html lang>`

**Files:** Modify `frontend/src/i18n/index.ts`.

- Import the 7 dictionaries; `dictionaries: Record<Locale, Dict> = { en, it, es, fr, de, pt, nl }`.
- Re-export `Locale` from `./locale`.
- `I18nProvider`: `useState(() => initialLocale ?? detectInitialLocale())` (keep optional
  `locale` prop for tests); `setLocale` persists + setState; effect sets
  `document.documentElement.lang`. Context value exposes `{ locale, t, setLocale }`.
- `useT(): Dict` unchanged. Add `useLocale(): { locale, setLocale, locales }` where `locales`
  = `SUPPORTED_LOCALES.map(l => ({ value: l, label: LOCALE_LABELS[l] }))`.

## Task 4 — Translated dictionaries (×6)

**Files:** Create `frontend/src/i18n/{it,es,fr,de,pt,nl}.ts`.

- Each: `import type { Dict } from "./en"; export const <lang>: Dict = { …translated… };`
- Structurally identical to `en.ts` (same nested keys, same order), only values translated.
- Terminology aligned with `backend/app/services/reporting/i18n.py`.
- Generated by one subagent per language (reads `en.ts`, writes the sibling file).

## Task 5 — `LanguageSwitcher` component

**Files:** Create `frontend/src/components/LanguageSwitcher.tsx`.

- Mantine `Select`: `data={locales}`, `value={locale}`, `onChange={v => v && setLocale(v as Locale)}`,
  `allowDeselect={false}`, `aria-label={t.common.language}` (add `common.language` key — see Task 6),
  compact width.

## Task 6 — Wire switcher into AppShell + LoginPage; add label keys

**Files:** Modify `frontend/src/components/AppShell.tsx`, `frontend/src/pages/LoginPage.tsx`,
and all 7 dictionaries (add `common.language` = native word for "Language").

- AppShell header: render `<LanguageSwitcher />` in the right-hand group, before the logout
  button.
- LoginPage: render `<LanguageSwitcher />` in the bottom centered group.
- Add `common.language` to `en.ts` first, then to each translated dict (build gate enforces
  parity, so this must be done in all 7).

## Task 7 — Tests

**Files:** Create `frontend/src/i18n/__tests__/i18n.test.tsx`.

- Deep-key parity: for each non-en dict, its flattened key set equals `en`'s.
- `detectInitialLocale`: localStorage wins; else `navigator.language`; else `en`.
- Switcher behavior: rendering with `locale="it"` shows the Italian string for a known key.

## Task 8 — Build gate + push

- `cd frontend && npm test && npm run build && npm run lint` — all green.
- Commit, push `feat/frontend-i18n`, open PR, wait for CI green, squash-merge.
